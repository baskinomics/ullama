# Agent Guidelines: Ullama

## 1. System Context & Constraints
Ullama manages a `llama.cpp` router server on a workstation with an NVIDIA RTX 4090 (24GB VRAM) and AMD Ryzen 9 7950X3D running CachyOS.
- **VRAM Strategy:** Only one model loaded at a time (`--models-max 1`) to prevent OOM.
- **CPU Affinity:** Bound to cores 0-7 (`taskset -c 0-7`) to optimize for the 3D V-Cache CCD.
- **Ports:** Router server on 8001; Open WebUI (Docker) on 3000.
- **llama.cpp source:** `~/workspace/machine-learning/llama.cpp` (outside this repo). `make build` pulls and rebuilds with CUDA.
- **HOST_ENV.md:** Auto-generated by `scripts/update_agent_context.sh` — don't edit manually.

## 2. Developer Commands
Use `make` for all tasks.

| Task | Command |
| :--- | :--- |
| **Build** | `make build` |
| **Update** | `make update` (same as build, pulls + rebuilds) |
| **Clean** | `make clean` (removes `llama.cpp/build/`) |
| **Run** | `make server` |
| **Run (debug)** | `make server-debug` |
| **Run (tmux)** | `make server-tmux` |
| **Stop** | `make stop` |
| **Docker up/down** | `make docker-up` / `make docker-down` |
| **Docker logs** | `make docker-logs` |
| **Port open** | `make port-open` |
| **Validate** | `make validate` (syntax + shellcheck) |
| **Lint** | `make lint` (shellcheck only) |
| **Test** | `make test` (bash syntax check) |

**Gotcha:** `make stop` calls `stop-server-tmux.sh` which uses `err()` (exits code 1) if the tmux session doesn't exist. This is expected behavior, not a bug.

## 3. OpenCode Session Setup
Read `opencode.jsonc` for configured models (5 total), provider endpoint (`http://jupiter:8001/v1`), and context/output limits.
- `.ignore` inverts `.gitignore` for logs — agent has read access to `scripts/logs/`.

## 4. Knowledge Base & Workflow
Git-native documentation system:

- **Decisions:** `docs/adrs/` — immutable architectural decisions.
- **Design:** `docs/specs/` — technical blueprints (see `docs/specs/git-native-docs.md` for taxonomy).

- **Architecture:** `docs/ARCHITECTURE.md` — design notes and decisions.

## 5. Coding Standards
### Bash
- `#!/usr/bin/env bash` + `set -euo pipefail`.
- `main() { ... }` pattern, called as `main "$@"` at end of file.
- `readonly` for constants, `local` for function variables.
- **Logging:** Use `scripts/utils/logging.sh` — `log()` (blue), `warn()` (yellow), `err()` (red, exits 1), `debug()` (cyan).

### Git
- Conventional Commits (`feat`, `fix`, `chore`, `docs`, `refactor`, `test`).
- Imperative mood, lowercase subject, no trailing period.
- Reference specs/ADRs in commits: `feat: ... (refs docs/specs/...)`.

## 6. Remote Access
- **Attach tmux:** `tmux attach -t ullama-server` (SSH to `jupiter` first).
- **API:** `http://jupiter:8001/v1`
- **Server logs:** `scripts/logs/server.log` (auto-rotated, keeps 5 archives).
- **tmux session name:** `ullama-server` (set in `start-server-tmux.sh`).

## 7. Presets
- **Linux (production):** `config/presets.ini` — INI format with `[*]` global defaults and per-model sections.
- **Linux (dev/experimental):** `config/presets-dev.ini` — more models, not loaded by default.
- **macOS:** `config/presets-macos.ini` — separate file with `DEFAULT` section and Metal acceleration.
- Models discovered automatically (no restart needed).

## 8. Key Architecture Notes
- **Router Mode:** Single `llama-server` instance handles all models via preset config. Dynamic loading/unloading.
- **tmux → systemd:** Current server runs via tmux (temporary). `docs/specs/systemd-plan.md` outlines the systemd migration. See ADR 0002.
- **Context limits:** ADR 0003 documents the context limit architecture to prevent OOM crashes. Ensure `limit.output` ≤ 25% of `limit.context` to avoid compaction loops.
